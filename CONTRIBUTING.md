# Contributing to Bunny Hop Adventure

Thank you for your interest in contributing to Bunny Hop Adventure! This document provides guidelines and instructions for contributing to this project during Hacktoberfest and beyond.

## Hacktoberfest Participation

This repository is participating in Hacktoberfest! To have your pull requests count towards Hacktoberfest:

1. Register at [hacktoberfest.com](https://hacktoberfest.com/)
2. Submit pull requests to this repository during October
3. Make sure your PRs follow our contribution guidelines
4. Wait for your PR to be reviewed and merged

## How to Contribute

1. **Find an Issue**: Look for issues labeled with `hacktoberfest` or `good-first-issue`
2. **Fork the Repository**: Click the "Fork" button at the top right of this repository
3. **Clone Your Fork**:
   ```bash
   git clone https://github.com/YOUR-USERNAME/bunny-hop-game.git
   cd bunny-hop-game
   ```
4. **Create a Branch**:
   ```bash
   git checkout -b feature/your-feature-name
   ```
5. **Make Changes**: Implement your feature or fix the bug
6. **Test Your Changes**: Make sure the game still works properly
7. **Commit Your Changes**:
   ```bash
   git add .
   git commit -m "Add feature: your feature description"
   ```
8. **Push to Your Fork**:
   ```bash
   git push origin feature/your-feature-name
   ```
9. **Create a Pull Request**: Go to the original repository and click "New Pull Request"

## Development Setup

1. No special setup is required - this is a simple HTML/CSS/JavaScript game
2. Open `index.html` in your browser to test changes
3. You can use a local server if you prefer:
   ```bash
   python -m http.server 8000
   ```

## Beginner-Friendly Tasks

If you're new to open source, consider these tasks:
- Add new obstacle types
- Implement power-ups
- Create different backgrounds
- Add sound effects
- Improve mobile controls
- Add difficulty levels

## Code Style Guidelines

- Use consistent indentation (2 spaces)
- Add comments for complex logic
- Follow existing naming conventions
- Keep functions small and focused

## Pull Request Process

1. Update the README.md if needed
2. Ensure your code works in major browsers
3. Link the PR to any related issues
4. Wait for review and address any feedback

Thank you for contributing to Bunny Hop Adventure!
